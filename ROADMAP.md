## 🏗️ 架构分层

Anima-AIOS 采用**双层架构**：

```
┌─────────────────────────────────────────┐
│      Anima-skill - 游戏化体验层          │
│      （快速迭代 · 用户驱动）             │
├─────────────────────────────────────────┤
│      Anima-core - 静默成长引擎           │
│      （稳定优先 · 问题驱动）             │
└─────────────────────────────────────────┘
```

**设计哲学：**
- **skill** 负责精彩（可视化、游戏化、交互）
- **core** 负责稳定（数据、算法、规则）

详见：[ARCHITECTURE_PRINCIPLES.md](docs/ARCHITECTURE_PRINCIPLES.md)

## 📊 版本发布节奏

```
2026-03  ────────● v5.0.0 (品牌升级 + skill 基础)
                 │
2026-04  ────●───┼───● v5.1.0 (skill: 成就 | core: 检索)
                 │
2026-05  ────●───┼───● v5.2.0 (skill: 挑战 | core: 数据)
                 │
2026-06  ────────●───● v5.3.0 (skill: 社交 | core: 算法)
                 │
2026-07  ────────●───● v5.4.0 (skill: 个性 | core: 性能)
```

**发布原则：**
- skill 和 core 独立版本号
- 大版本对齐（v5.x）
- skill 发布频率高于 core
- core 发布需充分测试
